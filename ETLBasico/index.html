ETL Básico com SQL

O processo de ETL (Extract, Transform, Load) é uma prática fundamental na integração de dados e na construção de data warehouses. Ele envolve três etapas principais:

1. Extração (Extract): Nesta fase, os dados são coletados de diversas fontes, como bancos de dados, arquivos CSV, sistemas ERP, entre outros. A extração é crucial para garantir que todos os dados relevantes sejam capturados para análise posterior.

2. Transformação (Transform): Após a extração, os dados brutos são transformados para atender aos requisitos de negócios. Isso pode incluir a limpeza de dados, a padronização de formatos, a remoção de duplicatas e a aplicação de regras de negócios. A transformação garante que os dados estejam em um formato adequado para análise.

3. Carregamento (Load): Na última etapa, os dados transformados são carregados em um data warehouse ou outro sistema de destino. O carregamento pode ser feito de forma incremental ou em lote, dependendo das necessidades do negócio. Esta fase garante que os dados estejam disponíveis para análise e relatórios.

O processo de ETL é essencial para a criação de uma base de dados consolidada e confiável, permitindo que as organizações tomem decisões informadas com base em dados precisos e atualizados.

A atividade proposta utiliza como fonte de dados os arquivos produtos, NCM e vendas, todos no formato .csv. Esses arquivos são essenciais para o processo de ETL, pois fornecem os dados brutos que serão extraídos, transformados e carregados no sistema de destino.
